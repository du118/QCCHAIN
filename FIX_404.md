# 🚨 官网 404 错误 - 自动修复指南

## 错误信息
```
404 There isn't a GitHub Pages site here.
```

## 原因
GitHub Pages **没有启用**，需要手动开启。

---

## ✅ 解决方案（2 选 1）

### 方案 A: 手动启用（推荐，5 分钟）

**这是最简单的方法**！

#### 步骤 1: 打开 GitHub 仓库
https://github.com/du118/QCCHAIN

#### 步骤 2: 进入 Settings
- 点击顶部 **"Settings"** 标签

#### 步骤 3: 找到 Pages
- 左侧菜单，向下滚动
- 找到 **"Pages"**（在 "Code and automation" 部分）

#### 步骤 4: 启用 Pages
**按顺序选择**：

1. **Source**: 选择 `Deploy from a branch`
2. **Branch**: 选择 `main`
3. **Folder**: 选择 `/ (root)`
4. 点击 **"Save"** 按钮

#### 步骤 5: 等待部署
- 等待 2-5 分钟
- 页面顶部显示 🟢 表示成功

#### 步骤 6: 访问官网
https://du118.github.io/QCCHAIN/

---

### 方案 B: 使用 Netlify 部署（备选，10 分钟）

如果 GitHub Pages 有问题，用 Netlify：

#### 步骤 1: 访问 Netlify
https://www.netlify.com/

#### 步骤 2: 注册/登录
- 使用 GitHub 账号登录

#### 步骤 3: 添加网站
- 点击 "Add new site"
- 选择 "Import an existing project"
- 连接 GitHub
- 选择 `QCCHAIN` 仓库

#### 步骤 4: 配置部署
- **Build command**: 留空
- **Publish directory**: `/`（根目录）
- 点击 "Deploy site"

#### 步骤 5: 获取官网链接
- Netlify 会生成一个链接
- 格式：`https://xxx-xxx.netlify.app`

## ⚠️ 常见问题

### Q1: 点了 Save 没反应？

**解决**：
- 刷新页面
- 重新选择一次
- 清除浏览器缓存

### Q2: 等待很久还是 404？

**解决**：
- 等待 10 分钟
- 访问 https://github.com/du118/QCCHAIN/actions 查看部署状态
- 如果有错误，截图发给我

### Q3: Settings 里没有 Pages 选项？

**原因**：
- 仓库是私有的
- 或者你是协作者不是所有者

**解决**：
- 确认仓库是公开的
- 或者使用 Netlify 方案
